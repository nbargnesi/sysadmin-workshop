Pipes

Pipes — Sysadmin Workshop


Pipes connect multiple commands together sending the output of one as input to another.

Under the covers, an operating system pipe is shared between the two programs. The first program writes to the pipe; the second program reads from it.

Example: who | wc -l

Filters

Filters — Sysadmin Workshop


Strictly speaking, filters are the elements that together form a command pipeline.

The command pipeline who | wc -l has two filters: who and wc.

Example:
find / -name '*.tar.[gz|bz2]' -exec ls -s "{}" \; | sort -n | vim -

Redirection

Redirection Part I — Sysadmin Workshop


Three files are typically open in every program. These are stdin, stdout, and stderr.

Input (stdin) is file descriptor 0. Output (stdout) is file descriptor 1. Error output (stderr) is file descriptor 2.

Demo: vim

Redirection Part II — Sysadmin Workshop


  • Redirecting stdin: <
  • Redirecting stdout: 1>;
  • Redirecting stderr: 2>

Basically two ways to do output redirection — M>N or M>&N. The former redirects file descriptor M to file N. The latter redirects file descriptor M to where file descriptor N currently points.

Demo: >&2, >>log 2>&1, echo foo >&2 | wc -l

Special Variables

Special Variables — Sysadmin Workshop


  • $1, $2, ... — positional parameters
  • $@ — positional parameters (array)
  • $# — number of positional parameters
  • $* — IFS expansion of positional parameters
  • $- — current shell options
  • $$ — process id of current shell
  • $_ — most recent parameter
  • $IFS — input field separator
  • $? — most recent foreground program exit status
  • $! — most recent background program process id
  • $0 — name of shell or script

Exit Codes

Exit Codes — Sysadmin Workshop


The $? special variable can be used to string together commands and control the execution of scripts.

Demo: touch with $?, inline if-touch, touch ||
